Device Info Privacy Policy

Device Info does not collect or store any user data on its own. The app is integrated with a few third-party SDKs that collect some data like crash logs, advertising ID, etc. to work in the required manner. We do not collect any personal data like email address, mobile number, name, address. The data collected by third-party SDKs is transferred and stored in an encrypted manner. The app requires a few permissions to work effectively. The sole purpose of these permissions is to help you in testing your device.

Required Permissions

  • USE_FINGERPRINT: Allows an app to use fingerprint hardware. It is used for Fingerprint Test.
  • USE_BIOMETRIC: Allows an app to use fingerprint hardware. It is used for Fingerprint Test (for higher APIs).
  • VIBRATE: Allows access to the vibrator. It is used for Vibration Test.
  • INTERNET | ACCESS_WIFI_STATE | ACCESS_NETWORK_STATE: It is used to show information in the Network Tab of the app.
  • BILLING: It is required for Remove Ads feature (billing purposes).
  • READ_PHONE_STATE: Allows read-only access to phone state, including the current cellular network information, the status of any ongoing calls, and a list of any PhoneAccounts registered on the device. It is used in Device Tab of the app.
  • CAMERA: It is used for the Flashlight Test.
  • MODIFY_AUDIO_SETTINGS: It is used in Ear speaker and Loudspeaker Test.
  • BLUETOOTH: Allows applications to connect to paired Bluetooth devices. It is used in Bluetooth Test.
  • BLUETOOTH_CONNECT: It is also used to connect to paired Bluetooth devices in Android 12+.
  • QUERY_ALL_PACKAGES: Allows the query of any normal app on the device, regardless of manifest declarations. It is required for showing app-related info.

We explicitly ask for some of these permissions. Some of them are automatically granted by Android. If you have any doubts regarding this privacy policy, please send us a message using the form below.
